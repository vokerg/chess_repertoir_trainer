# Project working guides

This directory contains detailed, human-readable project working guides. It is not GitHub's agent-skill directory. Executable agent workflows live under `.github/skills/*/SKILL.md` and may link back to these guides.

- [API feature module](./api-feature-module.md): work within the API's current mixed module/legacy structure.
- [Frontend feature](./frontend-feature-module.md): keep Angular work feature-local and avoid growing page components.
- [Contracts](./contracts.md): maintain API boundary shapes without mixing in domain or persistence logic.
- [Architecture review](./architecture-review.md): check project ownership and dependency boundaries.

Current architecture belongs in `docs/architecture.md` and `docs/frontend/`. See `docs/README.md` for the canonical documentation index.
